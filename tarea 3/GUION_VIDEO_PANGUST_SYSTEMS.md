### ⏱️ LA ETAPA DE DISEÑO (1:30 - 2:45)

#### Slide 4: ¿QUÉ ES LA ETAPA DE DISEÑO?

```
ELEMENTOS A MOSTRAR:
- Definición formal
- Posición en el ciclo de vida (mostrar Pangust Systems aquí)
- Frase clave: "De requisitos a soluciones técnicas"

NARRACIÓN (45 segundos):
"La etapa de DISEÑO es el primer paso en la fase de desarrollo 
de cualquier sistema de información.

Se define como: 'El proceso de aplicar distintas técnicas y principios 
con el propósito de definir un dispositivo, proceso o sistema con 
los suficientes detalles como para permitir su realización física'.

Para Pangust Systems, esto significa: Cuando un cliente pyme nos presenta 
sus requisitos —por ejemplo, 'necesito automatizar mis procesos de ventas 
y proteger mi información de clientes'— la etapa de diseño es donde 
nuestro equipo técnico especifica CÓMO vamos a lograrlo.

¿Qué base de datos usaremos? ¿Qué arquitectura en la nube? 
¿Cuáles serán los interfaces de usuario? ¿Cómo aseguramos los datos? 

Todas estas decisiones se toman en la ETAPA DE DISEÑO."
```

#### Slide 5: IMPORTANCIA PARA PANGUST SYSTEMS

```
ELEMENTOS A MOSTRAR:
- Gráfico de costo de cambios en el ciclo de vida
- Beneficios clave (bullets)
- Comparación de costos: cambio en diseño vs cambio en mantenimiento

NARRACIÓN (45 segundos):
"¿Por qué esta etapa es TAN importante para nosotros en Pangust Systems?

PRIMER PUNTO: Evita costos desastrosos. Si detectamos un error 
en la fase de diseño, cuesta casi nada corregirlo. Pero si ese error 
llega a producción después de que el cliente ya está usando el sistema, 
la reparación es exponencialmente más cara, afecta su operación 
y daña nuestra reputación.

SEGUNDO PUNTO: Garantiza seguridad desde el inicio. Nuestro valor central 
es 'Compromiso con la seguridad'. En el diseño establecemos 
cómo protegeremos la información de nuestros clientes pymes.

TERCER PUNTO: Permite estimaciones realistas. Con un buen diseño, 
podemos decir exactamente cuánto tiempo toma, cuántos recursos necesitamos 
y cuál será el costo para el cliente.

CUARTO PUNTO: Facilita la comunicación. El cliente ve mockups, diagramas 
y especificaciones claras, no solo promesas."
```

#### Slide 6: DISEÑO LÓGICO vs DISEÑO FÍSICO

```
ELEMENTOS A MOSTRAR:
- Comparación lado a lado
- Iconos diferenciados
- Ejemplos de Pangust Systems para cada uno

NARRACIÓN (35 segundos):
"El DISEÑO se divide en dos componentes que se complementan:

DISEÑO LÓGICO (El QUÉ):
Es independiente de tecnología. Responde: ¿Qué debe hacer nuestro sistema 
para una pyme? ¿Cuáles son los procesos? ¿Cómo fluye la información?
Usamos diagramas de flujo de datos y casos de uso.

DISEÑO FÍSICO (El CÓMO):
Es específico a tecnología. Responde: ¿Con qué herramientas lo hacemos? 
¿Qué servidor? ¿Qué base de datos? ¿Qué lenguaje de programación?
Usamos especificaciones técnicas detalladas.

Para Pangust Systems, ambas son necesarias. El cliente necesita entender 
el QUÉ, pero nuestro equipo técnico debe dominar el CÓMO."
```

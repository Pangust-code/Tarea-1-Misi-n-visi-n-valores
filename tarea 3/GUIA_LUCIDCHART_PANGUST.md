# Guía práctica para usar Lucidchart en el video de Pangust Systems

## Objetivo
Usar Lucidchart para mostrar la parte práctica de la etapa de diseño en el video: primero el diseño lógico, luego la arquitectura física y finalmente las interfaces.

## Qué deben hacer en el video
1. Presentar el problema de la pyme.
2. Explicar que la etapa de diseño convierte requisitos en una solución técnica.
3. Usar Lucidchart para dibujar la solución.
4. Mostrar tres cosas:
   - Diagrama de datos.
   - Arquitectura del sistema.
   - Interfaces básicas.
5. Cerrar explicando cómo eso resuelve el problema.

## Paso a paso en Lucidchart

### 1. Crear el documento
- Entrar a Lucidchart.
- Crear un diagrama nuevo en blanco.
- Ponerle un nombre como: "Pangust Systems - Diseño del sistema".

### 2. Dibujar el diagrama de datos
Incluyan estas entidades:
- Cliente
- Vendedor
- Producto
- Venta
- Detalle_Venta

Relaciones sugeridas:
- Cliente realiza Ventas.
- Vendedor registra Ventas.
- Venta incluye Productos.
- Venta tiene Detalle_Venta.

### 3. Dibujar la arquitectura
Incluyan estos bloques:
- App móvil o web.
- Servidor o API.
- Base de datos.
- Nube o hosting.
- Bloque de seguridad.

La idea es mostrar cómo viaja la información desde el usuario hasta la base de datos.

### 4. Dibujar las interfaces
Hagan pantallas simples como:
- Login.
- Menú principal.
- Registrar venta.
- Ver clientes.
- Ver productos.

### 5. Exportar o capturar
- Exporten los diagramas como imagen.
- O graben la pantalla mientras los construyen.
- Luego metan esas imágenes en la presentación o en el video.

## Qué decir mientras muestran Lucidchart
- "Aquí estamos haciendo el diseño lógico."
- "Aquí definimos qué datos necesita el sistema."
- "Ahora pasamos al diseño físico, donde vemos cómo se implementa."
- "Estas interfaces muestran cómo lo usará el usuario final."
- "Así resolvemos el problema con diseño profesional."

## Recomendación de orden para el video
1. Portada.
2. Misión y visión de Pangust Systems.
3. Problema de la pyme.
4. Explicación de la etapa de diseño.
5. Lucidchart: diagrama de datos.
6. Lucidchart: arquitectura.
7. Lucidchart: interfaces.
8. Conclusión.

## Consejo práctico
No intenten hacer muchos diagramas. Con estos tres alcanza para demostrar que entienden la etapa de diseño y que saben aplicarla al caso de Pangust Systems.
